面向新用户与开发者

CodingGirl：带可视化工作流与超级 Trace 的 coding agent / AI IDE 原型 目标是把模型的工作状态“变得可监督、可回放、可恢复”。

这套文档以“能用、能改、能继续迭代”为标准组织内容：先跑起来，再理解核心对象与工具语义， 然后再进入 P2/P3 的可生产化路线。

开发约束

单文件尽量不超过 800 行；达到 1000 行必须拆分。

前后端要同步推进：后端能力必须能在 UI 或用户路径里被消费。

上手路线

建议阅读与运行顺序

  1. 1) 快速开始 跑通 Python core 与桌面端启动方式，确认环境与依赖没问题。
  2. 2) 使用与工作流 理解 run / tool / diff / approval / artifact 在 UI 里怎么呈现。
  3. 3) 架构与代码导览 从对象模型与工具边界入手，再看 indexer、repo map 与 runtime 事件。
贡献方式

如何让改动有价值

优先改“成功率”

  • 工具参数校验、失败恢复、可回放证据链。
  • 减少脏 diff（CRLF/LF、patch 稳定格式）。

优先改“可解释性”

  • 事件语义单一事实源，UI 只做投影。
  • 关键失败要能定位、能复现、能恢复。

同步前端适配

  • 新增工具/索引能力要有 UI 入口或可见面板。
  • 避免“后端很强，但用户用不到”。
