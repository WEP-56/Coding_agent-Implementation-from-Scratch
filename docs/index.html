Reality Check

已经实现什么，尚缺什么What exists already, and what is still missing.

当前已经落地 / Implemented now

  • Python dataclass contracts: Task, Plan, ToolCall, ToolResult, Event.
  • 安全文件边界：RepoWorkspace 拒绝绝对路径与越界访问。
  • Unified diff patch 应用、备份、冲突拒绝与校验。
  • SQLite 运行记录、tool_call/event 记录链。
  • Python indexer：manifest、symbols.sqlite、repo_map。
  • 桌面端 session、timeline、diff、artifact、approval、terminal、trace export。

当前仍是裂缝 / Still a split system

  • Python orchestrator 仍然很窄，目前只覆盖 replace-style 目标格式。
  • Desktop runtime 与 Python runtime 还没有统一成一个 canonical engine。
  • 上下文压缩、resume、fork、replay 只在桌面端形成了部分雏形。
  • 索引能力主要偏向 Python，尚未真正统一覆盖 TypeScript / Rust 主工作流。
  • “像产品一样工作”的体验已出现，但底层运行语义还没有完全统一。
GitHub Pages

部署方式很简单Deployment stays intentionally simple.

为什么适合 Pages / Why Pages fits

这套文档不依赖 Jekyll、VitePress 或 Docusaurus。只要 GitHub 能托管静态文件，它就能发布。

The docs do not depend on Jekyll, VitePress, or Docusaurus. If GitHub can host static files, the site can ship.

最小发布步骤 / Minimal publishing steps

  1. 把仓库推到 GitHub。
  2. 在仓库设置中启用 GitHub Pages。
  3. Source 选择当前分支的 /docs 目录。
  4. 首页入口就是 docs/index.html。
