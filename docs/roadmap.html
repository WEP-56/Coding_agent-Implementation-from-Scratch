Usage Tutorial

学会怎么用，而不是只会打开目录 Learn the working paths, not just the folder names.

这页关注“如何正确体验仓库中的现有能力”。你会看到 Python core 的验证路径、桌面端的交互路径，以及哪些行为更接近真正的 coding agent 产品。

This page focuses on how to use the current repository correctly: the Python validation path, the desktop interaction path, and which parts already resemble a real coding-agent product.

Best Entry / 最优入口

第一次看仓库，先跑 Quick Start；第二次再看这一页。

Use Quick Start on your first pass, then return here on your second pass.

Mindset / 阅读心态

把它看成“教学中的双轨工程”：Python 负责 core，桌面端负责产品体验原型。

Treat it as a two-track teaching project: Python handles the core, while the desktop app explores product-level UX.

Recommended Path

建议上手顺序A practical path for learning the repo.

  1. 1. 先确认 Python 测试可过 / Confirm the Python tests first. 这一步验证底层工具链没有马上断掉。 This validates that the lower-level toolchain is not immediately broken.
  2. 2. 生成索引 / Generate the index artifacts. 这样你能看见 manifest、symbols.sqlite、repo_map 这些“仓库理解层”到底是什么。 This lets you inspect the manifest, symbols.sqlite, and repo_map artifacts that form the repository-understanding layer.
  3. 3. 再开桌面端 / Start the desktop app next. 桌面端更接近真实工作流：会话、模式、timeline、approval、artifact 都在这里。 The desktop app is closer to a real workflow: sessions, modes, timeline, approvals, and artifacts all live here.
  4. 4. 最后再看深层原理 / Read the deep-dive after you have touched the system. 先体验，再分析，理解会更稳定。 Touch the system first, analyze it second. The design will make more sense.
Python Workflow

如何体验 Python coreHow to experience the Python core in a meaningful way.

建议动作 / Recommended actions

  1. 跑 pytest 看基础能力是否稳定。
  2. 跑 codinggirl init 初始化 sqlite。
  3. 跑 codinggirl index 观察索引产物。
  4. 跑 codinggirl llm-probe 验证 provider 适配。
  5. 最后再试 codinggirl orchestrate。

别跳过的观察点 / Observations worth keeping

  • RepoWorkspace 如何限制路径逃逸。
  • SQLiteStore 如何记录 run、tool_call、event。
  • patch_apply_unified_diff 如何处理备份、冲突和删除限制。
  • review_patch 和状态机为什么那么保守。
Desktop Workflow

如何体验桌面端How to exercise the desktop path.

1. 加入仓库

在左侧选择或添加 repo，让 session 真正绑定到工作区。

Add or select a repository so the session is actually workspace-bound.

2. 创建会话

新建 session，给一个具体任务标题。

Create a session with a concrete task title.

3. 选模式并发消息

用 Plan / Build / Auto 切换你的风险与自动化程度。

Use Plan / Build / Auto to control risk and automation.

4. 看右侧面板

重点盯 timeline、diff、tools、artifacts、trace 和 context debug。

Watch the timeline, diff, tools, artifacts, trace, and context debug panels.

为什么桌面端更像产品 / Why the desktop path feels more product-like

因为它已经有会话、turn、timeline、approval、rollback、terminal、context compaction、intent routing 等结构，而且这些结构会被真实界面消费。

Because it already has sessions, turns, timeline projection, approvals, rollback, terminal execution, context compaction, and intent routing, and those structures are exercised by a real interface.

Modes

模式不要混着理解Do not flatten the different permission models into one thing.

入口 / Surface 模式 / Modes 含义 / Meaning
Python / Telegram readonly, write, exec 偏后端权限模型，直接决定能否写文件或执行命令。
Desktop plan, build, auto 偏产品协作模型，控制 mutating tools 是否需要审批、是否自动应用。

两套模式现在仍然没有统一成一个 canonical permission grammar，这正是后续演进重点之一。 The two mode systems are still not unified into one canonical permission grammar.

Common Mistakes

常见误区Mistakes readers make when they skim the repo.

把 UI 当成全部

桌面端能打开不代表 core 已经统一完成。

A working UI does not mean the core runtime is already unified.

把 orchestrate 当成成熟 agent

它是 MVP 链路验证器，不是完整任务规划器。

It is an MVP chain validator, not a full task planner.

忽略 trace 与 artifact

真正有教学价值的是“系统怎么留下运行证据”。

The real learning value is in how the system leaves trace and artifact evidence behind.
