Python Core

最短命令链Smallest useful Python command sequence.

py -m pip install -e .[dev]
py -m pytest
py -m codinggirl init
py -m codinggirl index --repo . --index-dir .codinggirl/index --max-lines 160
py -m codinggirl llm-probe --provider mock --model mock-1 --prompt "hello adapter"

为什么是这 5 步 / Why these five steps

  1. pip install 安装包和测试依赖。
  2. pytest 快速验证 workspace、patch、sqlite、indexer、policy、adapter 主链没有坏。
  3. codinggirl init 初始化本地 sqlite schema。
  4. codinggirl index 生成 manifest、symbols.sqlite、repo_map。
  5. llm-probe 验证 LLM adapter 通道本身可用。

看结果时重点观察 / What to look for

  • .codinggirl/codinggirl.sqlite3 是否生成。
  • .codinggirl/index/ 是否出现 manifest.json、symbols.sqlite、repo_map.txt。
  • 测试是否覆盖了 workspace、patch、sqlite、llm adapter 等基础能力。
Narrow Orchestrator

当前 orchestrate 的真实边界The current orchestrator is intentionally narrow.

可以这样试 / You can try it like this

py -m codinggirl orchestrate ^
  --repo . ^
  --db .codinggirl/codinggirl.sqlite3 ^
  --goal "replace [old] with [new] in [README.md]"

不要误解它 / Do not over-read it

它现在不是通用 agent planner，而是一个用来打通 plan → read → diff → review → apply → verify 链路的 MVP。

It is not a general-purpose planner yet. It is an MVP that proves the plan → read → diff → review → apply → verify chain.

Pages Deployment

GitHub Pages 发布Publish the docs site with no extra build step.

  1. 确保 docs/ 已提交到仓库。
  2. GitHub 仓库设置中开启 Pages。
  3. Source 选择当前分支的 /docs 目录。
  4. Pages 生效后，入口页就是 docs/index.html。

这套站点是静态文件，不需要额外构建命令。 The site is plain static files, so no extra docs build step is required.
