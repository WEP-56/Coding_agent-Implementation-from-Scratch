最短命令链

先跑通，再理解 先确保环境与依赖没问题，再进入架构细节。

本页默认在仓库根目录执行命令（Windows 以 py 为例）。

Python Core 桌面端
建议顺序

1) Python core → 2) index_build → 3) 桌面端（Web 或 Tauri）。

依赖提示

只读文档不需要 Node/Rust；运行桌面端需要 Node；构建 Tauri 桌面壳需要 Rust/Tauri 环境。

Python Core

安装与验证

安装（开发模式）

py -m pip install -e .[dev]

如果没有 py，可用 python 代替。

跑测试（推荐）

py -m pytest

这一步验证 workspace、patch、sqlite、indexer、policy 等底层链路是否可用。

索引与 Repo Map

生成可查询的仓库理解产物

CLI 索引（已有命令）

py -m codinggirl index --repo . --index-dir .codinggirl/index --max-lines 300

会生成 manifest.json、symbols.sqlite、repo_map.txt。

工具化索引（用于 runtime）

index_build

桌面端与后端桥接后，可以直接以工具调用形式触发索引更新。

Desktop

启动桌面端

Web 预览（最快）

cd apps/desktop
npm install
npm run dev

适合先看 UI 与交互，不依赖 Rust。

Tauri 桌面壳

cd apps/desktop
npm install
npm run desktop

需要本机 Rust/Tauri 环境。构建产物：npm run tauri build。

启动后建议检查什么

  • workflow / trace / tools 面板是否能展示 run 的事件与工具调用。
  • 审批（approval）路径是否工作；是否仍存在轮询（后续要替换为事件推送）。
  • diff、artifact、rollback 元数据是否能对齐到同一 run。
