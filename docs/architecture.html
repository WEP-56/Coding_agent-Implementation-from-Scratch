Deep Dive

从概念到实现，把系统拆开看 Break the system down from concepts to concrete implementation.

这页不是字面意义上的“零基础教学”，而是面向已经会写代码、想真正理解 coding agent 怎么搭的人。我们会从对象模型、运行时、工具层、前端状态、Tauri 命令和现有架构裂缝，一层层拆开。

This is not a literal beginner tutorial. It is for builders who already code and want to understand how a coding agent is assembled: object model, runtime, tools, frontend state, Tauri commands, and the current architectural seams.

Core Thesis / 核心判断

决定 coding agent 成败的，不只是模型，而是对象边界、工具语义、上下文生命周期和用户可观察性。

The success of a coding agent depends not just on the model, but on object boundaries, tool semantics, context lifecycle, and user-visible observability.

Current Tension / 当前张力

Python core 已经很适合讲“引擎”，桌面端已经很适合讲“产品”，但两者还不是同一套权威运行时。

The Python core is already good for teaching the engine, and the desktop app is already good for teaching the product, but they are not yet the same authoritative runtime.

Python Core Walkthrough

Python 线怎么搭起来How the Python line is assembled.

1. Contracts 先行

contracts.py 定义 Task、Plan、PatchSet、ToolCall、ToolResult、Event。

This file defines the canonical dataclass objects before any tool or UI layer is introduced.

2. Workspace 限定边界

runtime/workspace.py 拒绝绝对路径、阻止 .. 逃逸、限制读取体积。

runtime/workspace.py rejects absolute paths, prevents .. escapes, and caps read size.

3. Tool 层尽量薄

builtins_fs.py 和 builtins_patch.py 把文件读写和 patch 语义包成统一 tool handler。

builtins_fs.py and builtins_patch.py expose file and patch behavior through simple tool handlers.

4. SQLite 记录运行

storage_sqlite.py 用 run、step、tool_call、event 记录执行过程。

storage_sqlite.py persists execution through run, step, tool_call, and event tables.

5. Repo 理解层

manifest.py 扫描文件、symbols.py 用 Python AST 抽符号、repo_map.py 做轻量排序。

manifest.py scans files, symbols.py extracts Python symbols via AST, and repo_map.py ranks useful entries.

6. Orchestrator 是 MVP

core/orchestrator.py 打通最短链路，但目标格式仍然窄，只适合 replace-style 示例。

core/orchestrator.py proves the shortest execution loop, but the goal grammar is still intentionally narrow.

Why Diff, Approval, Rollback Matter

为什么这三件事必须一起讲Why diff, approval, and rollback belong together.

Diff 是事实单元

只要系统想改代码，就应该留下 diff 或等价的 mutation provenance。没有这个层，就做不了 review、artifact 或回滚。

Whenever the system mutates code, it should leave a diff or an equivalent mutation provenance object. Without that layer, review, artifacts, and rollback all become weak.

Approval 与 Rollback 不是附属功能

审批决定“能不能做”，回滚决定“做错了怎么办”。这两者都依赖统一的变更对象和 trace 记录。

Approval answers “may we do this?”, while rollback answers “what happens when it goes wrong?”. Both depend on a unified mutation object and trace record.

Current Fault Lines

当前架构的关键裂缝The main seams in the current architecture.

双运行时

Python core 有自己的状态机、sqlite 和 tool runner；桌面端有自己的 session turns、events、timeline 与命令层。

The Python core has its own state machine, sqlite storage, and tool runner; the desktop app has its own session turns, events, timeline projection, and command layer.

双对象体系

Python 里的 Task/Plan/Event 与桌面端的 SessionTurn/SessionEvent 还没有收敛。

The Python objects Task/Plan/Event still do not converge with the desktop objects SessionTurn/SessionEvent.

索引未共享

Python indexer 已经能产出 manifest / symbols / repo_map，但桌面端还没有直接消费这层结果。

The Python indexer already produces manifest / symbols / repo_map artifacts, but the desktop runtime does not yet consume them directly.

模式语义未统一

readonly/write/exec 与 plan/build/auto 都合理，但它们还不是一套 canonical grammar。

readonly/write/exec and plan/build/auto both make sense, but they are not yet one canonical grammar.

If You Build Your Own

如果你也想从零搭一个Rules worth stealing if you build your own agent.

  1. 先有 canonical objects，再谈模型与界面。 Define canonical objects before you optimize prompts or visuals.
  2. 第一天就做 repo-scoped safety。 Build repo-scoped path safety and command boundaries from day one.
  3. 让每次变更都能被审查、回滚、追溯。 Every mutation should be reviewable, rollbackable, and traceable.
  4. 上下文管理要产品化，而不是临时拼字符串。 Context management should become a product capability, not a string-concatenation hack.
  5. 在 UI 里暴露系统状态，而不是把不确定性藏起来。 Expose system state in the UI instead of hiding uncertainty behind chat bubbles.
