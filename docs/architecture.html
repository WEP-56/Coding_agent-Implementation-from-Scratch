核心判断

决定系统稳定性的不是提示词，而是：工具语义、失败恢复、证据链与单一事实源。

工具层落点

常用工具与建议用途

类别 工具 说明
读取 fs_read_range / fs_read_many_files 大文件与多文件切片读取，用于上下文组装与 hit 聚合。
发现 fs_list_files / fs_glob 仓库导航入口，支持默认 ignore 与自定义过滤。
搜索 search_rg scoped search（rg 优先，python fallback）。
编辑 fs_replace_text / fs_insert_at_line / fs_write_file 结构化编辑，减少脆弱 patch；写回保留 CRLF/LF。
patch patch_apply_unified_diff 支持 dry-run、结构化冲突、git diff 头、rename-only。
索引 index_build / index_query_repo_map / index_query_imports 生成/查询 manifest + symbols + repo_map，并支持 snippet。
执行 cmd_run exec 权限工具（高风险），后续需要 approvals/sandbox 收口。
并行 parallel_execute 多 Agent 并行执行，支持任务分解、依赖分析与结果合成。
技能 skill_load 按需加载 Skills 内容，支持 ShareAI 规范与 XML 包装。
索引与 repo map

如何把“仓库理解”接入用户路径

当前已具备

  • manifest 增量扫描与默认 ignore。
  • symbols.sqlite：Python + TS/JS（轻量）。
  • repo_map 查询：过滤、分组、可选 snippet。

仍需收尾

  • trust folders 与策略分层（系统/用户/项目）。
  • 桌面端面板消费 repo_map/imports（可跳转到文件+行号）。
  • 索引质量升级（TS/JS 精度、结果聚合与去重）。
