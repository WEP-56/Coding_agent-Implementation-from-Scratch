# CodingGirl 前端实现 TODO List（MVP）

> 基于 `frontend-design.md` 的工程拆解  
> 目标：Windows 首发，Tauri + React + Tailwind + shadcn/ui

---

## Phase A — 前端工程基建

- [ ] 初始化 `apps/desktop`（Tauri + React）
- [ ] 接入 Tailwind + shadcn/ui
- [ ] 建立主题系统（light/dark）
- [ ] 建立路由框架（工作台/仓库管理/设置/插件市场）
- [ ] 建立状态管理（建议 Zustand）
- [ ] 建立 API 层（对接现有 Python runtime，可先 mock）

**验收**
- [ ] 应用可启动，路由可切换，主题可切换

---

## Phase B — 信息架构与主框架（壳）

- [ ] 实现三栏布局骨架
  - [ ] 左栏：仓库切换 + 会话列表 + 全局会话入口
  - [ ] 中栏：会话流 + 时间线容器
  - [ ] 右栏：Tab 容器（Diff/Tools/Logs/Artifacts）
- [ ] 实现顶部状态条（当前仓库、会话模式、运行状态）

**验收**
- [ ] 三栏比例在 1366x768 下可用
- [ ] 能通过 mock 数据完整显示一个 run

---

## Phase C — 仓库管理（Repo-first）

- [ ] 本地仓库导入（文件夹选择器）
- [ ] 仓库列表（最近打开、固定）
- [ ] 仓库基础信息展示（路径、索引状态）
- [ ] GitHub/OAuth 入口占位 UI（不可用态说明）

**验收**
- [ ] 可导入并切换多个本地仓库
- [ ] 切换仓库后会话列表正确隔离

---

## Phase D — 会话与时间线

- [ ] 会话创建/切换/删除
- [ ] 会话模式切换（Plan/Build/Auto）
- [ ] 中栏消息流组件
- [ ] 时间线组件（等待/运行中/成功/失败）
- [ ] 关键事件通知（完成/失败/审批）

**验收**
- [ ] 会话模式切换能影响后续审批行为 UI
- [ ] 失败节点可见三动作入口（重试/回滚/诊断）

---

## Phase E — 右栏详情（核心）

### Diff Tab
- [ ] Split 默认 + Unified 切换
- [ ] 文件树导航 + hunk 跳转
- [ ] Apply / Reject / Rollback 操作栏

### Tools Tab
- [ ] ToolCall 卡片（入参/出参/耗时/状态）

### Logs Tab
- [ ] 日志流查看器（可复制）

### Artifacts Tab
- [ ] 产物列表（patch、report、index等）

**验收**
- [ ] 右栏 4 Tab 与同一 run 联动一致
- [ ] Diff 操作反馈到时间线和状态条

---

## Phase F — 审批门与安全交互

- [ ] 按模式实现审批逻辑：
  - [ ] Plan：不可 Apply
  - [ ] Build：默认审批后 Apply
  - [ ] Auto：自动 Apply（显示强提醒）
- [ ] 敏感操作弹窗（Ask + 记住本仓库）
- [ ] 一键回滚当前批次

**验收**
- [ ] readonly 风格路径不会出现可误触的写操作按钮
- [ ] Auto 模式可回滚且可追溯

---

## Phase G — 设置页与插件市场占位

### 设置页
- [ ] 基础设置（主题、默认仓库、通知）
- [ ] 安全权限（默认 mode、敏感操作策略）
- [ ] 模型配置（provider/model/base_url/key 占位与校验）

### 插件市场占位
- [ ] 占位目录页
- [ ] 本地插件导入（目录/zip）
- [ ] 已导入插件列表（启用/禁用仅UI态）

**验收**
- [ ] 设置项可持久化
- [ ] 插件导入流程可走通（即使不执行插件）

---

## Phase H — 可访问性与体验打磨

- [ ] 对比度检查（AA关键项）
- [ ] 焦点可见性与键盘可达（至少执行控制）
- [ ] 字号缩放与布局稳定性
- [ ] 空态与错误态文案完善

**验收**
- [ ] WCAG AA 关键项通过人工检查清单
